### About

I'm a backend engineer who enjoys the quiet parts of software — clean schemas, predictable APIs, and systems that don't fall over at 3 AM. Currently interning at **McKinley Rice**, where I work on [Redrob's](https://redrob.ai) GTM platform: a lead discovery and enrichment product used by sales and growth teams.

Most of my day is spent thinking about Elasticsearch queries, NestJS service boundaries, MongoDB indexes, and the occasional Puppeteer script that absolutely refuses to cooperate.

### Featured Projects

**Redrob GTM — People Search & Enrichment** · `NestJS` `Prisma` `MongoDB` `Elasticsearch`
<br>
Full-text search over large LinkedIn lead datasets with filters (title, company, location, keywords), pagination, and enrichment tracking for email and phone. Optimized index mappings and query structures for responsive search at scale.

**Redrob LLM — MCP-Enabled Chat Service** · `NestJS` `PostgreSQL`
<br>
Integrated Model Context Protocol domains into the chat microservice, enabling standardized tool workflows across Google Drive, Calendar, Gmail, and People Search. Added structured logging, retries, and end-to-end testing for tool invocation and failure handling.

**LinkedIn Lead Scraping & Ingestion** · `Node.js` `Express` `MongoDB` `Puppeteer`
<br>
Backend pipeline that scrapes LinkedIn profiles and search results, extracts structured fields, and ingests them into MongoDB for downstream enrichment. Built with retries, error handling, and logging for reliability.

**BidSmart — Online Bidding System** · `Java` `Spring Boot` `Spring Security` `MySQL`
<br>
REST API for a real-time auction platform with role-based auth (Buyer, Seller, Admin), bid validation, automated winner selection, and a clean layered architecture backed by JPA/Hibernate.
